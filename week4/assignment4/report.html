Introduction

This report is about usability and accessibility of the website we created for the course web technology. Our website is very limited regarding accessibility. Unfortunately the development of our application wasn't focus on both usability and accessibility. Nonetheless we were asked to evaluate how well our website scored in such aspects. We did took some aspects of accessibility into account: the website presents semantic elements such as the HTML tag strong. Some decisions made also ended up having a positive impact regarding accessibility: everything is quite well distinguishable on the website. The colors used differentiate well between the background and the actual content. When it comes to usability our website is also somewhat well visible on browsers of different sizes, but this is true only because we content we present is quite simple. There is a lot to improve such as start using media queries to make the website truly adaptable.

WCAG

The WCAG guidelines that are the most relevant to our page are guideline 1.1, guideline 1.4.1 and guideline 2.4. Guideline 1.1 is relevant to our page because we have a lot of images which is non-text content, and we provided all these images with text alternatives. Guideline 1.4.1 is relevant to our page because we did not implement this correctly, the font color (green) of the links to the Apple, Samsung, and Huawei websites is not a good combination with the background (dark blue and white marble). Guideline 2.4 is very relevant for our webpage because first we only met 1/4 of the guidelines from 2.4 and now after we executed the improvement plan, we meet all four the guidelines from 2.4.

Improvement plan

There are some improvement actions that we can do to improve our page, in terms of accessibility and usability (based on Appendix A) but not that much. We have listed the improvement actions below:

  1. We have to change the color of the links from Apple, Samsung and Huawei.
  2. We have to change the title to something that says more about the content of our website.
  3. We should add the ability that people can add an alt for an image in the Top selling models table.
  4. We should add links at the top of the page to each area of the content, to bypass blocks of content.
  5. When somebody clicks a link to another website, this site should open in a new tab.

1. We did not change the color, instead we added a beautiful dark gray navigation bar, with the links to Apple, Samsung and Huawei. We changed the original links to Test1, 2 and 3 So it is still visible what we changed.

2. Change the title into J&Y phones.

3. We can add the ability that people can add alt for an image in the top selling models table.

4. We added a dropdown menu in the navigation bar, called sections. In this menu you can quickly go to important sections of the webpage.

5.We added target="_blank" to the links after the URL and this causes that a link is opened in a new tab.

Appendix A

Principle Guideline are the requirements for success criteria level A met? brief explanation why our page complies or fails to comply to the given guideline.
Principle 1 1.1.1 Non-text content ✓ The page meets the requirements for success criteria level A, for guideline 1.1.1 because, All non-text content that is presented to the user has a text alternative that serves the equivalent purpose.
1.2 Audio Control N/A There is no video or audio on the webpage.
1.3.1 Info and Relationships ✓ This guideline has been fulfilled when it comes to semantics by using the tag strong in stead of bold. This allows the different clients to render the information that needs more attention in the most efficient way. However our website does not fulfill this criterium regarding the presentation of information because the website doesn't take every aspect of presentation in to account when the browser window's size changes.
1.3.2 Meaningful Sequence ✓ We don't have content for which the sequence is important, except for the headline. Due to the position of the headline in the html code it is guaranteed to be placed in the right position by all browsers. This guideline is thereby met.
1.3.3 Sensory Characteristics ✓ We meet this guidline. We do not identify content only by color, location or shape.
1.4.1 Use of Color x We do not meet this criterium because the links to Apple, Samsung and Huawei may be hard to see due to wrong choice of color.
1.4.2 Audio Control N/A There is no audio on our webpage.
Principle 2 2.1.1 Keyboard x A visual impaired person will probably be able to navigate through the entire webpage, but we haven't tested whether this is indeed the case. Hence this requirement hasn't been fully met.
2.1.2 No Keyboard Trap ✓ We do not have any plugins nor subsections on our website. There are not keyboard traps and hence navigation will never become trapped in some area of our page.
2.4.1 Bypass Blocks x We do not meet this guideline because we do not have a mechanism to bypass blocks of content.
2.4.2 Page Titled x Our webpage has a title, but this title doesn't describe the webpage. So we do not meet this guideline.
2.4.3 Focus Order x We do not meet this guideline because we do not have any focusable components except for the use of strong once.
2.4.4 Link Purpose (In Context) ✓ We meet this guideline because the purpose of each link can be determined from the link text alone or from the link text together with its programmatically determined link context.
