Self-evaluation report

Source 1: Web Content Accessibility Guidelines

Principle 1: Perceivable - Information and user interface components must be presentable to users in ways they can perceive.

Guideline 1.1

Guideline 1.1.1 Non-text content

The page meets the requirements for success criteria level A, for guideline 1.1.1 because, All non-text content that is presented to the user has a text alternative that serves the equivalent purpose.

Guideline 1.2

We don't have any video or audio on the webpage.

Guideline 1.3

1.3.1 Info and Relationships

This guideline has been fulfilled when it comes to semantics by using the tag strong in stead of bold. This allows the different clients to render the information that needs more attention in the most efficient way. However our website does not fulfill this criterium regarding the presentation of information because the website doesn't take every aspect of presentation in to account when the browser window's size changes.

1.3.2 Meaningful Sequence

We don't have content for which the sequence is important, except for the headline. Due to the position of the headline in the html code it is guaranteed to be placed in the right position by all browsers. This guideline is thereby met.

1.3.3 Sensory Characteristics

We meet this guidline. We do not identify content only by color, location or shape
